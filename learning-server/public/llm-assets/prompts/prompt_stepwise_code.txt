Workflow (strict order)

0) Confirm the teacher

3) Create a Lesson

7) Gather child background

8) Create the Lesson section

- Always use POST; it will overwrite the existing lesson section content.

9) Now we need to produce exercise.js, which is JavaScript code that will generate exercises on-the-fly
- the code would expose exactly one method - generateExercise(count=5), which will return a json in the following structure:
[{type, question_html, options:[], answer, formula_html, steps:{step_html, type, options:[], answer}}]
- type - "mcq" or "fib" - mcq are multiple choice and fib are fill-in-the-blanks (fib only allows numeric answers. no text)
- question_html - html formatted string with all key points underlined and all numbers in deep red and "what is asked" in bold
- options - array of upto 4 answers, each pure string (not html)
- answer - the answer to the question 
- formula_html - html formatted formula which will leaed to the answer
- steps - array of steps the student should follow to reach the answer eventually
- step_html - html formatted string in question format, just the way question_html is
- type - for steps, "mcq" is the only type
- options - upto 4 options, one of which is the answer for the step

Notes for generating exercises
- Every Concept identified in step 6 would have a category of differently worded but identical formula question
- Hence, in exercise.js, every Concept will have a function to itself, that will effectively evaluate the formula
- The function will accept multiple numeric inputs and multiple text fragments to compose the json structure defined above
- All question_html, formula_html and step_html should be valid english sentences, both gramatically and conceptually
- So, exercise.js would basically have a bunch of concept-functions, each accepting numeric values and text fragments
- generateExercise(count) would call these functions randomly, and will provide random numerics as params
- The text-fragments part would be derrived from the background info provided in step 7 and will be stored in string arrays, again provided randomyly for the question
- Eventually, the question, answer and steps would look like an LLM has generated it 

How to evaluate the correctness of the code
- Questions in the ixl report can be generated through these concept-functions
- Answer to these questions are also provided in the ixl report
- So, you must test each concept-function you have generated against this set and ensure that the result of the function execution and ixl answer matches
- You must also ensure that folliwng the steps eventually lead to the final answer
